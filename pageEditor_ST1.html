Splash screen

Introduction

The purpose of this document is to provide an overview of the design and development of the splash screen for the "Healthy Life" website. The splash screen serves as an introductory page that briefly welcomes users to the site and provides a loading effect before redirecting them to the home page.

The goal was to create a simple yet visually appealing splash screen that includes a title, a subtitle/motto, group member names, and a loading effect. I aimed to keep the design minimal and functional to ensure a smooth user experience.

Technical Description

The HTML file was structured using fundamental tags to achieve a clear and organized layout:

  • H1 Tag: Used to display the website's title: "Healthy life - Living Your Healthiest Life."
  • H2 Tag: Used for the website's subtitle or motto, providing a clear introduction to the site's mission.
  • IMG Tag: Applied indirectly as a background image covering the full height and width of the page.
  • UL and LI Tags: Used to list the names of the group members responsible for creating the project.
  • Meta Tag: A meta refresh tag was used to automatically redirect the user to the home page (home.html) after 4 seconds.

Styling and Layout

The splash screen includes a loading effect designed using only CSS. This loader visually indicates that the page is preparing content and then automatically redirects the user to the home page.

The splash screen’s design and layout were primarily achieved using CSS, ensuring that the page was simple yet visually appealing. I used flexbox to structure the content, allowing for easy alignment and centering of elements within the splash screen. The background image was applied using the background-image property with background-size: cover to ensure it filled the entire viewport while maintaining its aspect ratio. Additionally, I used the background-attachment: fixed property to keep the image static while scrolling.

A custom CSS loader was created to provide a subtle animation effect when the page loads. This was achieved using a linear gradient mask with an animation defined by keyframes. The keyframes gradually expanded the background size, simulating a loading effect. Furthermore, I used a semi-transparent background overlay to keep the text readable despite the background image. The text content was centered vertically and horizontally using flexbox with justify-content: center and align-items: center. I also applied a simple unordered list to display the names of group members without using bullet points by setting list-style: none. Overall, the styling was kept minimal, clean, and easy to navigate.

Accessibility

While developing the splash screen, I made sure to implement accessibility best practices to ensure inclusivity for all users. I included appropriate heading tags (H1 and H2) to maintain a clear and meaningful content hierarchy, which benefits screen readers and improves overall page structure. For the background image, I avoided embedding critical content within the image itself to prevent it from being missed by assistive technologies. The text content was placed in a high-contrast color over a semi-transparent background to ensure readability.

Additionally, I applied the alt attribute for the logo and icons on the splash screen to provide alternative text descriptions, allowing visually impaired users to understand the visual elements of the page. I also ensured that the page had no flashing or complex animations that could potentially trigger seizures in sensitive individuals. The loader animation was kept subtle and did not cause any motion-related issues. Lastly, I verified that the color contrast between text and background complied with WCAG 2.1 standards, ensuring all text was clearly legible.

Volunteer page

Introduction

The development of the Volunteer page was an enjoyable and rewarding experience, particularly as it reminded me of several similar projects I have worked on over the past seven years as a Front-End Developer. My goal was to create a page that was both simple and stylish while ensuring an engaging and user-friendly experience.

The overall design of the Volunteer page follows a clean and structured approach. One of the key elements I focused on was the volunteering program cards, as I found designing them to be the most engaging part of the process.

Technical Description

Volunteering Program Cards

Each volunteering program card features an image at the top, providing an engaging visual introduction to the program. Below the image, I placed an H3 heading tag for the program title, maintaining consistency with the page’s heading hierarchy. To enhance the card’s layout, I wrapped the content below the image within a div container and applied display: flex, ensuring a visually appealing and well-structured design. Additionally, I incorporated icons within each card to highlight key aspects of the volunteering programs, improving clarity and readability for users.

Reviews Section

The reviews section was implemented below the volunteering program cards and includes a review display area alongside a form for submitting new reviews. The reviews are presented using elementary cards, ensuring a clean and structured appearance. Each review card contains essential elements such as the reviewer’s name, score, feedback text, and date. To facilitate user interaction, a review submission form was included, utilizing display: grid for a well-structured and user-friendly layout.

Styling and Layout

For styling and layout, I leveraged a combination of CSS classes to precisely design and structure each element on the page. Flexbox was extensively used to style most components, ensuring proper alignment and spacing. The form section, however, was structured using a grid layout to create an intuitive and organized input structure. Additionally, consistent theming was applied throughout the page, maintaining visual harmony across elements by using uniform typography, spacing, and color schemes.

Accessibility

To ensure accessibility, I structured the page using proper HTML elements and a clear heading hierarchy. The form section includes appropriate labels for input fields, making it easier for users to navigate and interact with. Additionally, visual hierarchy was maintained through appropriate use of headings and content separation, improving readability and usability.

Also, I made sure that all images have descriptive alt text, ensuring they are accessible to users who rely on screen readers. Furthermore, I added an aria-label to the form so that assistive technologies can correctly identify and interpret its purpose. Finally, I checked that the overall text-to-background contrast followed WCAG (Web Content Accessibility Guidelines) standards. To achieve this, I used the WAVE Evaluation Tool to analyze elements across the page and made necessary adjustments to enhance readability and usability for all users.

Content Page

Introduction

The development of the Content page was guided by the need to maintain consistency with the website's overall design while ensuring an engaging and structured layout. The page was designed to present informative content in a visually appealing and well-organised manner.

The Content page begins with a banner/leaderboard at the top, which includes an H1 title and an accompanying image. This section serves as an introduction to the page’s subject and aligns with the website’s branding. The page follows a structured layout with different sections, each containing a title, text content, and a relevant image. This approach ensures a logical flow of information, making the content easy to read and navigate.

The main topic of the page revolves around the importance of sleep and recovery in maintaining a healthy life. The content explores how sleep impacts various aspects of well-being, including fitness, mental health, and nutrition. Additionally, a dedicated section provides practical tips to improve sleep quality, enhancing the page’s informative value.

Technical Description

To achieve a well-balanced and visually appealing layout, I used a combination of CSS classes to design and style each element precisely. The leaderboard and content sections were styled using display: flex, ensuring proper alignment and responsiveness across different screen sizes. The use of flexbox allows the text and images to be well-structured and adaptable to various devices.

One of the key features of this page is the inclusion of an aside section that serves as a sidebar menu. This menu contains quick navigation links to different sections of the content. To enhance usability, I applied the position: sticky technique, ensuring that the sidebar remains visible while scrolling. This feature provides users with continuous access to navigation options, improving the overall user experience.

Accessibility

In terms of accessibility for the content page, I ensured that all images have descriptive alt text, making them readable for screen readers. The form elements and navigation links were structured with appropriate aria-label attributes where necessary, ensuring that assistive technologies can interpret them correctly. Additionally, I verified the text-to-background contrast using the WAVE Evaluation Tool, making adjustments to ensure compliance with WCAG standards and improve readability for all users.

Challenges and Lessons Learned

Overall, working on the pages I had to make, I encountered no significant challenges.

I had to research how to make a loader element for the splash page using only CSS, which was interesting.

It was also interesting to study the subject on which our website was based, which is health-related topics and assets.

Compliance

The pages I have created are fully compliant with Janet's regulations.

I have ensured accessibility by adhering to the WCAG 2.1 guidelines.

Furthermore, the website is free of any inappropriate, offensive, or illegal content, and it does not collect or store user data since it lacks the necessary functionality.
