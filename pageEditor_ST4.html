Sitemap Page

Technical Description

The sitemap is built using SVG (Scalable Vector Graphics) elements, this structure allowes for a visually interactive representation of the different sections of website and pages to be connected in a way to show where which connection has been done in a more visual way throughuot the entire website solution. There are a couple of tags that have lots of importance; one of these are the SVG tags, they play an important roll in definning the overall drawing where all of the nodes, lines and the text are placed. It uses the Viewbox="0 0 1000 600" attribute, which is a coordinate system wheere (0,0) is the top left corner and (1000,600) is the bottom right. This allows for a scalable vector graphics that can adjust to different screen sizes without loosing its clarity.


Importance of the SVG in the sitemap : The SVG element is a crucial bevause it enables precise positioning of each sitemap node and connection. Unlike the traditional HTML element like div, SVG allows elements to be placed based on a coordinate system rather than relying on CSS Layouts. This ensures that the node and lines maintain a consistent structure regardless of the viewport size. It alos provides better scalability and interactivity as nodes and line can be dynamically updated or animated using CSS and Javascript however in this soution I have not used Javascript only a bit of CSS has been implemented.


Lines, Nodes, and Text: Building the Sitemap Structure:

The "line tag" elements play a crucial role in visually connecting different sections of the sitemap, forming a clear hierarchical structure. Each "line tag" is positioned using x1, y1 (starting point) and x2, y2 (ending point), effectively linking parent and child pages. These lines ensure that users can easily follow the structure of the website, understanding how pages are grouped and related. Without these connections, the sitemap would appear as disjointed elements, making it harder to interpret the site’s layout. Additionally, their styling (stroke: black; stroke-width: 2;) ensures they are clearly visible while maintaining a professional and organized look. The sitemap mimics real-world navigation by guiding the user’s eye from primary to secondary pages, improving usability and site comprehension.

The "rect tag" elements serve as clickable nodes, visually representing each page in the sitemap. By defining their x and y positions within the "svg tag", they are strategically placed to maintain an intuitive structure. The rounded corners (rx="20") create a more polished and user-friendly design, differentiating the nodes from rigid, square elements. These nodes are styled with fill: #6A0DAD to maintain visual consistency, and the hover effect (fill: #9C27B0) provides feedback when interacted with, improving the user experience. Since they are wrapped inside "a" tags, the "rect tag" elements also function as navigation buttons, allowing users to seamlessly move between pages. This transforms the sitemap from a static reference tool into an interactive navigation aid, making it highly functional.

Each node is labeled with a "text tag" element, ensuring that users can easily recognize the purpose of each section. Positioned using text-anchor="middle" and dominant-baseline="middle", the text remains perfectly centered within its corresponding rectangle. The styling (font-size: 14px; font-weight: bold; fill: white;) ensures maximum readability, making the labels stand out against the dark background of the nodes. Without these text elements, users would have to guess the purpose of each node, reducing the effectiveness of the sitemap. The "text" tag plays a vital role in clarity and accessibility, ensuring that the sitemap remains easy to read and navigate, even for users unfamiliar with the website structure.


Accessibility

In this page the accessibility has been put in measure by making sure that a tags have been assigned so that users can click on the relevant pages they would like to go to on the users end and also ensure a hover effect on all of the nodes have been put so that all sorts of users can have an easy to use and navigation, which is ease for all users no matter what age they are. This sitemap page has been designed with accessibility in mind by incorporating screen reader support, clear visual hierarchy, and interactive elements. The use of "a" tags around the nodes ensures that screen readers can recognize them as navigational links, allowing users with visual impairments to access different pages. The addition of aria-label attributes provides clear descriptions of each link’s destination, improving usability for assistive technology users. Additionally, the large, well-spaced nodes and connecting lines create a structured layout, ensuring users can visually follow the site's hierarchy without confusion. These considerations make the sitemap both functional and accessible, providing an intuitive navigation experience for all users.

Content Page

Technical Description

Explain the HTML and CSS techniques used to implement this page. Justify the logic and purpose of the key tags and styling choices focusing on functionality and design without copying large code sections. Explain the purpose of key HTML tags and CSS properties. Specify key HTML tags and CSS properties used to create structure and styling. The content page I designed is one of the most important pages in my project, focusing on the Importance of Good Health and Wellbeing, a topic I chose from the 13 UN Goals that I support. To build this page, I carefully used a variety of HTML and CSS elements to ensure both functionality and professional design. Some of the key tags I implemented include section tags, div tags, list elements, video and iframe elements for embedding media, containers, and the use of classes and IDs to structure and style the content effectively.

I used external CSS to keep the styling separate from the HTML document, making the code more organized and easier to maintain. The IDs were particularly useful for classifying different sections and enabling smooth navigation within the page. One of the standout features I incorporated is a side navigational panel, which is fully responsive and allows users to jump to different sections effortlessly. Additionally, I used containers to position multiple images side by side in a landscape format, ensuring a cleaner and more professional look rather than stacking them vertically, which could appear cluttered and less visually appealing.

The webpage was structured using key HTML elements such as "section", "div", "nav", lists, and media elements to create an organized and visually appealing design. The "section" tags were used to divide the page into meaningful sections, improving readability and making it easier to apply specific styles to different parts of the content. "div" elements were used as containers to group related content together, ensuring proper alignment and layout control. This was especially important for positioning images and text in a structured manner. Additionally, unordered "ul" and ordered "ol" lists were incorporated to present key points clearly, making the content more digestible for users. For multimedia integration, "video tags" and "iframe tag" elements were used to embed videos, enhancing user engagement by providing visual content directly on the page.

Containers were implemented to neatly arrange multiple images, preventing them from stacking awkwardly and instead displaying them in a landscape orientation for a professional look. Classes and IDs played a crucial role in styling and functionality; classes allowed for consistent styling across multiple elements using external CSS, while IDs were used for targeted styling and smooth navigation. The side navigation panel, built with the "nav tag" element and linked to section IDs, made the page more interactive by enabling quick access to different sections. Overall, these elements contributed to a well-structured, visually cohesive, and user-friendly webpage.


Accessibility

Accessibility was a key consideration in designing this page to ensure that all users, including those with disabilities, could navigate and interact with the content effectively. One of the main accessibility features implemented was semantic HTML, which helps screen readers and assistive technologies understand the structure of the page. The use of "section", "nav", and "div" elements ensured that content was organized logically, making it easier for assistive technologies to interpret. The navigation panel was designed to be keyboard-friendly, allowing users to navigate through sections using the tab key and ensuring proper focus states for interactive elements. In addition, I made sure that my content page was accessible by adding hover effects and making it fully responsive with a reasonable sizing for all type of reader to access the content with a variety of images so that user does not get bored.
